# External Integrations

**Analysis Date:** 2026-06-18

## Overview

This framework does **not** contain a running application or server. Integrations are
declared as agent tool capabilities and framework conventions ‚Äî they are invoked at
agent-runtime when the framework is used to develop AL extensions.

---

## Azure DevOps

**Purpose:** Central task management for all AI-assisted AL development.

**Usage:**
- Work item read/write via `ado/*` tool namespace (declared in `al-auto-dev.agent.md`)
- Work item comments posted after every agent run (template in `.github/instructions/azure-devops.instructions.md`)
- Work item tags set/read: `ai:auto`, `ai:plan-only`, `ai:implement`, `ai:review-only`, `ai:blocked`, `ai:done`, `bc-al`
- Branch naming convention tied to Work Item IDs: `ai/wi-{id}-kurzer-titel`
- Commit message format: `WI #{id}: kurze Beschreibung`
- Draft PRs created referencing `AB#{id}` (Azure Boards link syntax)

**Auth:** Not configured in this repo ‚Äî expected to be present in the agent runtime environment.

**Instruction file:** `.github/instructions/azure-devops.instructions.md`

**Agent tool namespace:** `ado/*` (declared in `.github/agents/al-auto-dev.agent.md` frontmatter)

## Business Central Server (OnPrem / SaaS)

**Purpose:** AL symbol source, build target, and test runtime for AL extension projects.

**Connection config (in sibling AL repos, not in this framework):**
- Config file: `.vscode/launch.json` ‚Äî required by `al-build-validation` skill
- Required fields: `server` (URL), `serverInstance`, `authentication`
- `environmentType`: `OnPrem` or `Sandbox`
- Authentication types: `UserPassword`, `Windows`

**Credentials policy:**
- Usernames and passwords are **never stored in files**. Users enter them in the VS Code authentication dialog at runtime.
- This policy is enforced in `.github/skills/al-build-validation/SKILL.md` and `.github/agents/al-build-tester.agent.md`

**AL LSP Tool calls (invoked by agents):**
| Tool | Purpose |
|------|---------|
| `al_downloadsymbols` | Downloads Base Application + System Application `.app` symbols into `.alpackages/` |
| `al_build` | Compiles the AL extension project |
| `al_getdiagnostics` | Retrieves compiler errors, warnings, and hints |
| `vscode_listCodeUsages` | LSP lookup for object usages in Base Application |

**Symbol storage:** `.alpackages/` ‚Äî populated at runtime; currently empty in framework repo.

## Authentication & Secrets Policy

**Hard rules (from `.github/policies/agent-policy.md` and `.github/copilot-instructions.md`):**
- Agents **must not** read, output, or store secrets, credentials, or customer data
- BC server credentials (username/password) are never written to files ‚Äî entered via VS Code dialog only
- No customer data may be used in agent prompts
- `launch.json` `server`/`serverInstance`/`authentication` fields are the only permitted connection metadata

**No auth providers configured in this repo itself** ‚Äî authentication context is provided by the runtime environment (VS Code Copilot, ADO service connections, Git credential manager).
